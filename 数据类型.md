<h3>整型</h3>
<ul>
	<li>如果不设置有无符号，默认有符号，如果想设置无符号需要添加unsigned关键字</li>
	<li>如果插入的数据超出了范围,mysql8.0及以上会直接报错，低于该版本会报异常且插入临界值</li>
	<li>如果不设置长度，会存在默认长度，长度的含义是指显示的长度，而非储存的长度（实际的储存长度在数据类型定义时就确定了），如果实际储存的数字长度小于设置的长度，会在数字前面自动填充0，但是不显示，如果要显示需要添加zerofill关键字，并且添加该关键字后类型会自动改为unsigned</li>
</ul>
<h3>小数</h3>
<ul>
	<li>m与d都可以省略
		<ol>
			<li>double跟float若省略则根据插入的数值来确定</li>
			<li>dec默认为m=10,d=0</li>
		</ol>
	</li>
	<li>定点型的精确度更高</li>
</ul>
<h3>字符型</h3>
<ul>
	<li>char与varcahr的区别在于，char(m)就固定了字节数，varchar(m)可以根据实际的数据（比如字节小于m时）自动调整分配空间，但char效率比varchar高</li>
</ul>
<h3>日期型</h3>
<ul>
	<li>date只能保存日期没有时间</li>
	<li>time只有时间没有日期</li>
	<li>year只保存年</li>
	<li>datetime保存日期+时间</li>
	<li>timestamp与实际时区有关，属性受mysql版本跟SQLMode影响</li>
</ul>
